# 에자일을 일상생활에 적용시켜보자

## 1. 일상 에자일이란

에자일 방법론을 꼭 IT 프로젝트에만 쓰라는 법은 없다.

일상은 프로젝트보다 더 **변수가 많고 계획이 자주 틀어지기에** 이에 맞는 ㄴ에자일 방법론을 연구해보았고 앞으로의 일상생활에 적용시켜 보기로 했다. 

그래서 **에자일을 일상에 적용한다면 어떤 모습일까** 생각해보고,

실제로 사용할 수 있는 형태로 개념을 정리해 보았다.

일상 에자일 이란, 거창한 계획을 세우는 대신

**작게 실행하고 → 결과를 확인하고 → 바로 수정하는 생활 방식**이다.

처음부터 완벽하게 맞는 방법을 찾으려 하기보다,

일단 시도해 보고 그때그때 맞게 조정하는 쪽에 가깝다.

### 그래서 어떻게?

- 계획보다 실행을 우선한다
- 한 번에 완성하려 하지 않는다
- 결과를 보고 계속 수정한다
- 반복할수록 방식이 정교해진다

### 예시

- 공부 계획을 세워도 그대로 지켜지지 않는 경우가 많다
    
    → 하루 실행 후 공부량을 조정한다
    
- 그림 연습 방법이 맞는지 확신이 없다
    
    → 일단 1주 해보고 부족한 부분만 보완한다
    
- 생활 루틴이 자꾸 무너진다
    
    → 시간을 줄이거나 순서를 바꿔본다
    


## 2. **핵심**

### 2.1 스프린트형 생활하기

에자일에서는 긴 계획보다 **짧은 실행 단위**를 중요하게 본다.

일상에서는 보통 **하루 또는 일주일 정도의 단위**가 적당하다.

너무 먼 계획을 세우면 중간에 상황이 바뀌기 쉽기 때문이다.

그래서 목표를 작게 나누어 **부담 없이 실행할 수 있는 형태**로 만드는 것이 중요하다.

### 예시 (1주 단위)

| 분야 | 목표 |
| --- | --- |
| 단어(일본어) | 단어 80개 |
| 독해(일본어) | 지문 10개 |
| 그림 | 인체(전신) 20개 |
| 포폴 | 캐릭터 1개 |

처음부터 완벽하게 맞추기보다

실행 가능한 수준으로 설정하는 것이 중요하다.


### 2.2 피드백 기록하기

에자일에서 중요한 것은 결과 자체보다

**결과를 보고 무엇을 수정할지 판단하는 과정**이다.

같은 방법을 계속 반복하면 같은 문제도 계속 반복된다.

그래서 실행 후에는 간단하게라도 기록을 남기는 것이 도움이 된다.

### 기록 예시

```
계획: 그림 2시간
실제: 1시간 정도 집중
문제: 구도 잡는 데 시간이 오래 걸림
수정: 참고자료 먼저 모아두기
```

이렇게 짧게라도 기록해 두면

다음 실행에서 같은 문제를 줄일 수 있다.


### 2.3 다양한 실험을 하고 개선하기

한 번에 맞는 방법을 찾기 어렵기 때문에

조금씩 수정하면서 맞춰가는 방식이 필요하다.

예를 들어

- 공부 시간이 너무 길면 줄여 보고
- 작업 속도가 느리면 단계를 나누어 보고
- 집중이 안 되면 환경을 바꾸어 본다

이 과정을 반복하면

점점 자신에게 맞는 방식이 만들어진다.

완벽한 방법을 찾는 것이 아니라

**점점 덜 틀리는 방법을 찾는 과정**에 가깝다.


### 2.4 우선순위 설정하기

할 일이 많을수록 무엇부터 해야 할지 헷갈리기 쉽다.

이때는 결과에 가장 큰 영향을 주는 작업을 먼저 하는 것이 효율적이다.

예를 들어 웹툰 준비를 한다면

- 브러시 설정보다
- 인체 비율 연습이 더 중요할 수 있다.

모든 일을 동시에 잘 하려고 하기보다

중요도가 높은 작업부터 처리하는 것이 부담을 줄인다.


## 3. 일상적으로 하는 법

### 3.1 목표를 지속 가능하게

며칠만 하고 멈추는 방법은 의미가 없다.

그래서 처음에는

**조금 부족한 정도의 작업량**이 오히려 유지하기 쉽다.

예시

- 하루 5시간 계획 → 부담 증가
- 하루 1~2시간 → 유지 가능


### 3.2 계획은 수정이 가능하게

계획은 예상대로 진행되지 않는 경우가 많다.

이때 계획을 포기하기보다

작업량이나 방식을 조금 조정해 보는 것이 도움이 된다.

예시

- 시간이 부족함 → 작업량 감소
- 집중이 어려움 → 작업 시간 분할


### 3.3 본인의 주제를 파악하기..

물론 자신을 믿는 것은 좋은 자세지만, 사람의 집중력은 오래 유지되기 어렵기 때문에

적당한 길이로 나누는 것이 효율적이다.

예시

- 50분 작업 + 10분 휴식
- 1시간 작업 후 짧은 정리

짧게 나누면 시작 부담도 줄어든다.


### 3.4 그냥 일단 하기

준비만 오래 하는 경우 실제 진행이 늦어질 수 있다.

자료 조사나 계획도 필요하지만

어느 정도 준비가 되면 실행을 시작하는 편이 도움이 된다.

실행하면서 부족한 부분을 발견하는 경우가 많기 때문이다.

또한, 준비만 오래하다보면 준비하는 척 하면서 다른 길로 샐 수 있다(나의 수많은 경험담)

## 4. 피드백은 어떻게?

### 기본적인 질문

- 오늘 가장 중요한 작업은 무엇인가
- 어느 정도 하면 완료로 볼 것인가
- 얼마나 시간이 필요한가

### 점검용 질문

- 실제로 계획한 만큼 진행되었는가
- 방해 요소는 무엇이었는가
- 다음에는 무엇을 바꿀 수 있는가

### 개선용 질문

- 시간이 부족할 때 줄일 부분은 무엇인가
- 예상보다 오래 걸린 이유는 무엇인가
- 다음 실행에서 개선할 점은 무엇인가


## 5. 왠지 앞으로 있을 것 같은 일들

- 처음부터 계획을 크게 세움
- 실행 기록을 남기지 않음
- 한 번 실패하면 중단함
- 완벽한 방법을 찾으려 함

이런 불상사가 생기지 않기 위해서 처음부터 계획 수립을 잘하고 기록을 꾸준히 해야겠다.

## 6. 시작 방법

에자일을 적용한다고 해서

특별한 도구가 필요한 것은 아니다.

간단하게는

1. 하루 목표를 정하고
2. 실행해 보고
3. 짧게 기록하고
4. 다음 날 조금 수정하는 방식으로 시작할 수 있다.

작은 변화라도 반복되면

점점 자신에게 맞는 방식이 만들어질 것이다.


## 7. 방향성 

처음부터 복잡하게 구성하기보다

최소한의 구조로 시작하는 편이 부담이 적을 것 같다.

예시

- 하루 1시간 실행
- 간단한 기록 작성
- 1주 후 방식 수정

이 과정을 반복하면서

조금씩 범위를 늘리면 성장이 더욱 쉬어질 것이다.


## 8. 결론 & 제언

다양한 방법론을 프로젝트 뿐만 아니라 일상에서 적용시키면 목표 성취를 더 효과적으로 할 수 있을 것 같다는 생각이 든다. 앞으로 이런 일상생활을 하여 학교 취업률에 이바지 하는 날이 오면 좋겠따.

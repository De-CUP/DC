# 비용산정기법(사업적 관점)
## 1. 비용 산정이란

프로젝트 수행에 필요한 **인력, 기간, 자원 비용을 예측하여 실행 가능한 계획을 만드는 과정**이다.

사업 관점 핵심:

- 예상 비용 = 곧 사업 리스크
- 비용 산정 정확도 = 프로젝트 성공 확률
- 계약 금액 산정의 기준

즉,

> 비용 산정은 기술 문제가 아니라 **경영 판단 도구**다.
> 

---

## 2. 비용 산정이 필요한 이유

1. 일정과 예산 오류 감소
→ 잘못된 견적은 손실로 직결
2. 프로젝트 규모 사전 파악
→ 인력 계획, 투자 규모 결정 가능
3. 발주자–수주자 간 신뢰 확보
→ 객관적 계약 근거 제공
4. 생산성 기준 확보
→ 저가 수주로 인한 품질 저하 방지

사업적 의미:

- 비용 산정 실패 = 일정 지연 + 품질 저하 + 손해 발생

---

## 3. 비용 산정에 영향을 주는 요소

### 3.1 프로젝트 요소

- 소프트웨어 복잡도
- 시스템 규모
- 요구되는 신뢰도 수준

복잡한 시스템일수록

→ 개발기간 증가

→ 비용 증가

---

### 3.2 자원 요소

- 인적 자원 (개발자 능력)
- 하드웨어 자원
- 소프트웨어 도구

사업 관점:

좋은 개발자는 비용이 비싸지만

→ 전체 개발기간 감소 가능

---

### 3.3 생산성 요소

- 개발자의 경험
- 개발 방법론
- 기술 수준

예:

고급 언어 사용 시 생산성 증가

→ 동일 기능을 더 빠르게 개발 가능

---

## 4. 주요 비용 결정 요소 (핵심 6가지)

1. 프로그래머 능력
2. 소프트웨어 복잡도
3. 소프트웨어 크기
4. 개발 가능 시간
5. 요구 신뢰도 수준
6. 기술 수준

사업적 해석:

> 비용은 기능 수가 아니라 난이도와 인력 수준에서 결정된다.
> 

---

## 5. 비용 산정 기법 종류

### 5.1 하향식 산정

전체 비용을 먼저 정한 후 세부 비용 분배

특징

- 빠른 추정 가능
- 초기 기획 단계에 적합

사업 활용:

- 투자 판단
- 예산 범위 설정

---

### 5.2 상향식 산정

작업 단위별 비용을 계산 후 합산

특징

- 비교적 정확
- 실제 개발 단계에 적합

사업 활용:

- 계약 금액 산정
- 일정 계획 수립

---

### 5.3 LOC (코드 라인 수) 기반 산정

코드 줄 수를 기준으로 비용 계산

예측 방식:

비관치 / 낙관치 / 기대치 평균 계산

예측 공식:

예측 LOC =

(낙관치 + 4×기대치 + 비관치) / 6

사업적 의미:

- 생산성 기준 설정 가능
- 개발 기간 예측 가능

---

### 5.4 개발 단계별 인월수 방식

각 단계별 투입 인력을 기준으로 비용 산정

특징:

LOC 방식보다 현실적

→ 실제 업무 분배 반영 가능

---

## 6. 사업 관점 핵심 정리

비용 산정 =

기술 문제가 아니라 **의사결정 시스템**

기업이 비용 산정을 하는 이유:

- 손해 보는 프로젝트 방지
- 일정 지연 방지
- 인력 낭비 방지
- 계약 분쟁 방지

결론:

정확한 비용 산정 능력 = 프로젝트 성공 확률 증가

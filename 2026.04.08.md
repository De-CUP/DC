# Map
키와 값으로 구성된 객체. 중복되는 키 값을 가질 수 없다. 
많은 데이터 중에서 원하는 데이터를 빠르게 찾을 수 있는 자료 구조

이럴 때 사용:
1. 정렬해야할 때
2. 많은 자료를 저장하고, 검색이 빨라야 할 때
3. 빈번하게 삽입, 삭제가 이루어지지 않을 때

# HashMap
키와 값을 이용하여 하나의 데이터로 저장한다
hashCode를 사용하기 때문에 예측할 수 없는 순서로 저장한다.
키는 중복이 되지 않지만 값은 중복이 될 수 있다.
해싱을 사용해 많은 양의 데이터를 검색할 때 성능이 좋다.

해시 함수

데이터의 효율적인 관리를 목적으로
임의의 길이를 가진 데이터를 고정된 길이의 데이터로 매핑하는 함수다.
매핑 전 원래 데이터의 값을 키, 매핑 후 데이터의 값이 해시라고 한다.
해시함수를 이용하여 데이터를 매핑하여 저장하고 검색하는 기법을 해싱이라고 한다.

# LinkedHashMap

입력된 키의 순서가 보장된다.
HashMap의 모든 기능을 사용하고, 이중 연결 리스트를 내부에 유지하여 입력된 자료의 순서를 보관한다.
LinkedHashMap = HashMap + LinkedList

HashMap의 저장된 값들을 순회하고자 할때 무작위로 출력되게 되는것을 보완하기 위해 등장하였다.
그래서 LinkedHashMap은 FIFO 구조로 데이터를 저장한다.

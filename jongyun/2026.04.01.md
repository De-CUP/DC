# GC란?

가비지 컬렉션은 자바의 메모리 관리 방법중의 하나로 JVM의 Heap 영역에서 동적으로 할당했던 메모리 중

필요 없게 된 메모리 객체를 모아 주기적으로 제거하는 프로세스이다.

![image.png](attachment:b8f4e9a2-9b5a-4e1c-b2c2-8a845addb153:image.png)

Java에서는 가비지 컬렉터가 메모리 관리를 대행해주기 때문에 Java 프로세스가 한정된 메모리를 효율적으로 사용할수 있게 하고, 개발자 입장에서 메모리 관리 메모리 누수 문제에서 대해 관리하지 않아도 되어 오롯이 개발에만 집중할 수 있다는 장점이 있다.

예를 들어 좀 미친 코드를 짠다면

```java
for (int i = 0; i < 10000; i++) {
  NewObject obj = new NewObject();  
  obj.doSomething();
}
```

루프문에 의해서 10000 건의 NewObject 객체는 for문 스코프 내에서 생성되고 사용되지만,

루프가 끝나고 루프 밖에서는 더이상 사용할 일이 없다.

만약 이런 객체들이 메모리를 계속 점유한다면, 다른 코드를 실행하기 위한 메모리 자원은 지속적으로

줄어들기만 할 것이다.

하지만 GC가 한번쓰이고 버려지는 객체들을 주기적으로 비워줌으로써 한정된 메모리를 효율적으로 사용할 수 있게 해주기 때문이다.

## 단점

자동으로 처리해준다고 해도 메모리가 언제 해제되는지 정확하게 알 수 없어 제어하기가 힘들며, 가비지 컬렉션이 동작하는 동안에는 다른 동작을 멈추기 때문에 오버헤드가 발생되는 문제점이 있다.

이를 Stop-The-World 라고한다.

```
**STW(Stop-The-World)**
GC를 수행하기 위해 JVM이 프로그램 실행을 멈추는 형상을 말하며,
GC가 작동하는 동안 GC 관련 스레드를 제외한 모든 스레드는 멈추게 되어
서비스 이용에 차질이 생길 수 있다. 
따라서 이 시간을 최소화 시키는 것이 쟁점이다.
```

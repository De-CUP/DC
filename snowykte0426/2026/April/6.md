### Kotlin Sealed Class
- Sealed Class의 특징 중 하나로 `when`을 이용할 때 모든 분기를 강제하도록 된다, 즉 `else`를 사용할 수 없도록 강제한다.
```kotlin
sealed interface Result<out T> {
    data class Success<T>(val data: T) : Result<T>
    data class Error(val message: String, val cause: Throwable? = null) : Result<Nothing>
    data object Loading : Result<Nothing>
}

fun handleResult(result: Result<User>) = when (result) {
    is Result.Success -> showUser(result.data)
    is Result.Error   -> showError(result.message)
    Result.Loading    -> showSpinner()
}
```
### Kotlin Flow
- `Mono`와 `Flux`와 거의 같으나 `Flow`는 **Cold**하다.
- **Cold Stream**이 가장 중요한 특성으로 구독이 있기 전까지 작동하지 않는다.
```kotlin
val flow = flow {
    println("시작")
    emit(1)
    emit(2)
    emit(3)
}

// 여기까지는 "시작"이 출력되지 않는다. 아직 아무것도 실행 안 됨.

flow.collect { value ->
    println(value)
}
// 출력: 시작 → 1 → 2 → 3
```
- `Flux`도 Cold하지만 Spring에선 Cold/Hot을 따로 구분할 이유가 많지 않았다.
```kotlin
// Repository
class MessageRepository {
    fun observeMessages(): Flow<List<Message>> = flow {
        while (true) {
            emit(db.getMessages())   // 현재 값 내보내기
            delay(1_000)
        }
    }
}

// ViewModel
class ChatViewModel : ViewModel() {
    val messages: StateFlow<List<Message>> = messageRepository
        .observeMessages()
        .stateIn(
            scope = viewModelScope,
            started = SharingStarted.WhileSubscribed(5_000),
            initialValue = emptyList()
        )
}

// UI
@Composable
fun ChatScreen(viewModel: ChatViewModel) {
    val messages by viewModel.messages.collectAsState()
}
```
- `stateIn()`에서 `SharingStarted.WhileSubscribed(5_000)`은 구독자가 사라진 후 5초가 지나면 업스트림도 멈추라는 의미다.
### Coroutine Scope
|Scope|용도|
|---|---|
|`viewModelScope`|Android ViewModel 내부|
|`lifecycleScope`|Android Activity/Fragment|
|`CoroutineScope(...)`|commonMain에서 직접 생성|
|`GlobalScope`|안티 패턴|

# CORS

SOP란?
브라우저의 기본 보안 정책이다.
같은 Origin(프로토콜 + 도메인 + 포트)에서만 리소스를 요청할 수 있다.

CORS란?
예외적으로 다른 Origin도 허용

동작방식

단순 요청

조건

> GET/POST/HEAD
> ContentType 제한있음

요청 `Origin: http://localhost:3000`

서버응답 `Access-Control-Allow-Origin: http://localhost:3000`

Preflight 요청

조건

> PUT/DELETE
> Authorization 헤더 있음
> ContentType:application/json

Preflight 요청 (Options)
```
OPTIONS /api/user
Origin: http://localhost:3000
Access-Control-Request-Method: POST
Access-Control-Request-Headers: Authorization
```

서버 응답
```
Access-Control-Allow-Origin: http://localhost:3000
Access-Control-Allow-Methods: POST, GET
Access-Control-Allow-Headers: Authorization
```
이걸 통과해야댐

CORS는 서버가 아닌 브라우저가 막는것

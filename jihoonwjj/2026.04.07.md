# MVCC

데이터베이스에서 동일 데이터에 대해 여러 버전을 유지함으로써 락 없이 동시성을 관리하는 기법

트랜잭션이 데이터를 읽을 때 해당 시점의 스냅샷을 제공하여 읽기 작업과 쓰기 작업이 충돌하지 않도록 한다.

MVCC의 구성 요소

1. Undo Log
- 트랜잭션 실행 이전 상태를 기록하여 롤백이나 MVCC를 구현한다.
- 이전 데이터를 참조할 수 있도록 데이터 변경 내용을 저장한다.
2. Redo Log
- 트랜잭션이 커밋된 후 장애 복구 시 변경 내용을 재적용한다.
- 데이터 무결성과 지속성을 보장한다.
- 마지막 커밋된 변경 사항을 복원한다.
3. Buffer Pool
- 메모리 내 데이터를 관리해 디스크 I/O를 줄이고 성능을 높인다.
- 디스크 접근 최소화

작동 방식

1. 스냅샷 읽기
- 트랜잭션 시작 시점의 데이터 버전을 읽는다.
- 다른 트랜잭션의 변경 작업은 현재 트랜잭션에 영향을 미치지 않는다.
2. 쓰기 작업
- 기존 데이터를 수정하지 않고 새로운 버전을 생성한다.
- 커밋되지 않은 쓰기 작업은 다른 트랜잭션에서 볼 수 없다.
3. Garbage Collection
- 참조되지 않는 오래된 데이터 버전을 주기적으로 삭제한다.

읽기 작업과 쓰기 작업 간 충돌을 최소화한다.
락 경합이 없어 높은 동시성을 제공한다.
Dirty Read, Non-Repeatable Read 문제를 해결한다.

Phantom Read 방지는 Serializable 수준에서만 가능하다.
트랜잭션이 오래된 스냅샷을 참조하여 최신 데이터와 불일치가 발생할 수 있다. <- 더 높은 격리 수준을 사용하거나, 애플리케이션 레벨에서 동기화를 처리한다.
두 트랜잭션이 동시에 동일 데이터를 업데이트하면 변경 사항이 일부 손실될 수 있다. <- 애플리케이션 레벨에서 낙관적 락 또는 비관적 락을 활용하여 충돌을 감지하고 처리한다.

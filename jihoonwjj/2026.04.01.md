# Race Condition 이란?

둘 이상의 입력 또는 조작의 타이밍이나 순서 등이 결과값에 영향을 줄 수 있는 상태

공유 자원을 둘 이상의 스레드 혹은 프로세스가 읽거나 쓰면서 결과값이 의도와 달라질 수 있는 문제

문제점
- 여러 스레드나 프로세스마다 실행 속도가 달라서 잘못된 값을 읽거나 수정할 수 있다
- 여러 스레드나 프로세스가 데이터를 수정 시 예기치 않은 상태로 데이터가 변경될 수 있다
- 여러 스레드나 프로세스의 각각 다른 실행 속도로 인해 실행 흐름을 읽기 힘들어 디버깅이 어려움
- Race Condition을 방지하기 위해 무분별한 락 사용시, 대기 시간으로 인해 성능 저하가 발생

예방법

- 공유 데이터에 접근하는 부분을 임계 영역으로 지정하고, 한 번에 하나의 실행 흐름만 해당 영역에 들어가도록 한다
- 데이터를 적절히 분리하여 각 스레드가 독립적으로 처리하도록 한다
- 공유 데이터를 수정하는 함수나 메서드를 스레드 안정성을 보장하여 구현한다

# Garbage Collection 이란?

자바의 메모리 관리 방법중의 하나로 JVM의 Heap 영역에서 동적으로 할당했던 메모리 중 필요없게 된 메모리 객체를 모아 주기적으로 제거하는 프로세스

Stop-The-World (STW)
가비지 컬렉터가 가비지들의 메모리 공간을 회수하기 위해 자바 애플리케이션의 실행을 멈추는 것
이러한 시간을 줄이는 것을 GC튜닝

동작 방식

Mark and Sweep

- Mark: Root Space로부터 그래프 순회를 통해 연결된 객체들을 찾아내어 각각 어떤 객체를 참조하고 있는지 찾아서 마킹한다.
- Sweep: 참조하고 있지 않는 객체들을 Heap에서 제거한다
- Compact: Sweep 후 분산된 객체들을 Heap의 시작 주소로 모아 메모리가 할당된 부분과 그렇지 않은 부분으로 압축한다

동작 과정

JVM 메모리는 객체의 생존 기간에 따라 물리적인 Heap 영역을 Young, Old로 나눈다

Young

- 새롭게 생성된 객체가 할당 되는 영역
- 대부분 금방 필요없어짐
- Eden 영역과 Survivor 영역으로 나누어 관리함
- 짧은 STW 시간 안에 메로리를 회수할 수 있도록 한다

Old

- Young 영역에서 살아남은 객체가 복사되는 영역
- Young 영역보다 크게 할당됨
- 가비지 적게 발생

1. 새로 생성된 객체들은 Eden 영역에 생성
2. Eden 영역이 가득 차면 Eden 영역에 있는 가비지들의 메모리 공간 회수 (Mark and Sweep)
3. Eden에서 살아남은 객체들은 survivor 영역으로 옮겨짐
4. survivor 영역이 가득차 다른 survivor 영역으로 옮겨지면 age값이 올라가는데 일정치 이상으로 올라가면 객체는 Old 영역으로 옮긴다(Promotion)

# 패킷분석

네트워크에서 오가는 패킷을 수집하고 해석하여
시스템이나 네트워크의 상태를 파악하는 기술이다.
패킷스니핑이라고도한다.

네트워크 특성 분석
사용자의 IP, MAC 파악
대역폭 사용량 및 피크타임 분석
보안 위협 및 악성 행위 탐지
설정 오류 및 병목 지점 확인

NIC의 무차별 모드
- NIC를 모든 패킷을 수신할 수 있는 모드로 설정한다.
- LAN 세그먼트 내 모든 트래픽을 수집 가능하다.

데이터 전송 흐름
1. 클라이언트가 `http://www.google.com` 요청
2. TCP -> 신뢰성 있는 전달을 위해 헤더 생성
3. IP -> IP 헤더 캡슐화
4. Ethernet -> MAC 주소 부여 후 물리층 전송
5. 서버가 SYN+ACK로 응답
6. 클라이언트 ACK -> 3-way 핸드세이크 완료
7. 클라이언트가 웹 페이지 요청 (GET)

공격 기법

# Macof 공격
- 가짜 MAC/IP 주소를 대량으로 만들어 스위치의 CAM 메모리를 오버플로우된다. (CAM: 실제 데이터 내용을 기반으로 초고속 검색을 수행하는 특수 메모리)
- 장비 기능 마비 -> 일종의 DoS 공격

# ARP Spoofing
- 타켓 MAC 주소를 해커의 MAC 주소로 위조한다.
- 스위치나 라우터가 해커에게 트래픽을 전달한다.

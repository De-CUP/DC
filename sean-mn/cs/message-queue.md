# Message Queue란

MQ(Message Queue)는 애플리케이션 간에 메시지를 비동기적으로 교환하기 위한 통신 방식 또는 그 구현체를 말한다.

## 구조

기본 구조는 간단하다. <br>
메세지를 보내는 보내는 쪽(Producer)이 큐에 메시지를 넣으면,받는 쪽(Consumer)이 큐에서 메시지를 꺼내 처리한다. <br>
두쪽이 동시에 동작하지 않아도 된다는 점이 핵심이다.

## 왜 사용하는가?

- **비동기 처리**: 요청자가 응답을 기다리지 않고 바로 다음 작업을 진핼할 수 있다.

- **결합도 감소**: 시스템 A가 시스템 B의 상태를 몰라도 메시지를 보낼 수 있다. B가 잠시 다운되어도 메시지는 큐에 쌓여 있다가 나중에 처리된다.

- **부하 분산**: 갑자기 요청이 몰릴 때 큐가 버퍼 역할을 해서 Consumer가 처리 가능한 속도로 작업을 소화할 수 있다.


## 사용 예시

- 이메일/알림 발송: 회원가입 요청 즉시 응답하고, 이메일은 큐를 통해 비동기 처리

- 주문 처리: 주문 접수 후 재고 차감, 결제, 배송 등 후속 작업을 순서대로 처리

- 마이크로서비스 간 통신: 서비스끼리 직접 API 호출 대신 메시지로 이벤트 전달

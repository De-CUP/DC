# 레디스(Redis)란?

Redis는 In-Memory 데이터 구조 저장소로, 데이터를 디스크가 아닌 RAM에 저장하여 초고속으로 I/O 기능을 제공하는 NoSQL 시스템이다. <br>
디스크 기반 DB보다 약 100배 이상 빠른 성능을 자랑하며, 단순한 Key-Value 저장소를 넘어 다양한 자료구조를 제공한다.

## 사용 사례
- **Cache** — 가장 대표적인 사용 사례. DB 조회 결과를 캐싱하여 부하를 줄인다
- **세션 저장소** — 로그인 세션 정보를 빠르게 읽고 쓴다
- **Pub/Sub 메시징** — 채널 기반의 발행/구독 메시징을 지원한다
- **분산 락 (Distributed Lock)** — 여러 서버 간 동시성 제어에 활용된다
- **리더보드** — Sorted Set을 활용하여 실시간 랭킹 시스템을 구현한다

<br>

## 지원 자료구조
 
| 자료구조 | 설명 |
|---|---|
| String | 가장 기본적인 Key-Value 형태 |
| List | 순서가 있는 문자열 목록 (Linked List) |
| Hash | 필드-값 쌍으로 구성된 맵 |
| Set | 중복 없는 문자열 집합 |
| Sorted Set | 점수(score) 기반으로 정렬된 집합 |
| Bitmap | 비트 단위 연산 지원 |
| HyperLogLog | 대용량 데이터의 중복 카운팅에 특화 |
| Stream | 로그성 데이터를 위한 메시지 스트림 |

<br>

## 영속성 (Persistence)
 
In-Memory 저장소이지만 서버 재시작 시 데이터가 유실되지 않도록 두 가지 영속성 옵션을 제공한다.
 
- **RDB (Redis Database)** — 특정 시점의 스냅샷을 디스크에 저장한다. 복구 속도가 빠르지만 스냅샷 사이의 데이터는 유실될 수 있다.
- **AOF (Append Only File)** — 모든 쓰기 명령을 로그로 기록한다. 데이터 유실 가능성이 낮지만 RDB보다 파일 크기가 크다.

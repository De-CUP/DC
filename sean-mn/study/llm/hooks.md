# Hook이란 - Claude Code

Hook은 영어로 '낚시 바늘'이나 '갈고리'를 뜻한다. <br>
이름 그대로 프로그램의 특정 동작이 일어나는 순간을 낚아채서, 우리가 원하는 명령을 실행시키는 기술이다.

<br>

클로드의 작업 흐름을 강물이라고 생각하면, Hook은 그 강물에 던져놓은 낚싯바늘이고, <br>
특정 이벤트(코드 생성, 수정, 실행 등)가 발생하는 순간 이를 가로채 추가 작업을 수행한다.

<br>

## Hook의 핵심 개념

Hook은 단순한 자동화가 아니라 이벤트 기반 실행(Event-driven execution)이다.

즉, 우리가 직접 실행하지 않아도 특정 시점에 자동으로 동작한다.

예를 들어:

- 코드가 생성된 직후
- 파일이 수정된 이후
- 특정 명령이 실행되기 전/후

이러한 “타이밍”에 개입해서 원하는 로직을 실행할 수 있다.

## Hook을 왜 사용해야 할까?

우리가 매번 똑같은 일을 지시하는 대신, 파일이 저장될 때마다 확실하게 작업을 자동화하여 실수를 줄이고 생산성을 극대화 시키기 위해 사용된다.

# Apache Kafka란?

아파치 카프타(Apache Kafka)란 대용량의 실시간 데이터를 빠르고 안정적으로 처리하기 위해 사용되는 분산 스트리밍 플랫폼이다.

쉽게 비유하자면, 여러 곳에서 쏟아지는 수많은 메시지를 중간에서 아주 빠르고 정확하게 분류해서 전달해 주는 '데이터 우체국'과 같습니다.

## 구성 요소

Kafka는 크게 세 가지 역할로 나뉜다.

- **Producer(생산자)**: 메시지를 만들어 Kafka에 보내는 주체다. 예를 들어 웹 서버에서 발생하는 로그가 여기에 해당한다.
- **Broker(브로커)**: 메시지를 받아서 저장하고 전달하는 Kafka 서버 자체를 말한다. 여러 대의 브로커가 클러스터를 이루어 동작한다.
- **Consumer(소비자)**: 브로커에 저장된 메시지를 읽어가는 주체다. 데이터를 분석하거나 저장하

## Topic과 Partition

메시지는 **Topic**이라는 카테고리 단위로 분류된다. 예를 들어 '주문 이벤트', '로그인 이벤트'처럼 주제별로 나눌 수 있다.

각 Topic은 여러 개의 **Partition**으로 나뉘어 저장된다. Partition 덕분에 데이터를 병렬로 처리할 수 있어 높은 처리량을 달성할 수 있다.


## 주요 특징

- **높은 처리량**: 초당 수백만 건의 메시지를 처리할 수 있다.
- **내구성**: 메시지를 디스크에 저장하기 때문에 시스템이 재시작되어도 데이터가 유실되지 않는다.
- **메시지 보존**: 일반적인 MQ와 달리, Consumer가 메시지를 읽어가도 일정 기간 동안 메시지가 삭제되지 않는다. 덕분에 여러 Consumer가 같은 메시지를 독립적으로 읽을 수 있다.
- **확장성**: 브로커를 추가하는 것만으로 수평 확장이 가능하다.

## 활용 사례

- 실시간 로그 수집 및 모니터링
- 결제, 주문 등 이벤트 기반 마이크로서비스 간 통신
- 클릭스트림, 사용자 행동 데이터 분석 파이프라인
- IoT 센서 데이터 처리

# .NET C# GC 동작 방식 정리

.NET의 GC는 CLR에서 제공하는 자동 메모리 관리자로, <br>
애플리케이션의 메모리 할당과 해제를 대신 관리하여 메모리 누수를 방지, 개발자의 부담을 줄여준다.

## 동작 단계
.NET의 GC는 기본적으로 Mark-Sweep-Compact 패턴의 세 단계에 거쳐 메모리를 정리한다.

- **Mark** : root(정적 필드, 스택 변수, CPU 레지스터 등)부터 시작하여 참조 체인을 따라가며 현재 사용 중인 '살아있는' 객체를 식별 후 표시한다.

- **Sweep** : 표시되지 않은 객체들은 가비지로 간주하여 메모리에서 제거한다.

- **Compact** : 살아남은 객체들을 모아 메모리 파편화를 제거하고 연속된 공간을 만든다. 이때 객체의 주소도 바뀌므로 루트의 포인터도 갱신된다.

## .NET GC의 비용 절감 방법
모든 메모리를 매번 검사하는 것은 비효율적임으로, ,NET은 객체의 수명에 따라 세 영역으로 나눈다.

- **0세대** : 새로 생성된 객체가 배치되며, GC가 가장 많이 발생한다. 대부분 임시 변수가 해당된다.

- **1세대** : 0세대 객체 중 살아남은 객체가 승격되는 버퍼 영역이다.

- **2세대** : 장기 생존 객체(정적 데이터 등)가 위치하며, 수집 비용이 가장 낮기에 낮은 빈도로 GC가 수행된다.

- **LOH (Large Object Heap)** : 85,000 바이트 이상의 큰 객체들이 할당되는 별도의 공간이다. 논리적으로 2세대에 속하며, 복사 비용의 문제로 Compact를 수행하지 않는다.

# [C#] var 타입의 자동 추론 방식

`var`는 변수를 선언할 때 타입을 명시하는 대신 컴파일러가 우변의 식을 분석하여 타입을 자동으로 결정하는 <br>
C#의 암시적 타입 선언 키워드이다.

> 런타임 동적 타입이 아니라 컴파일 타임 정적 타입 추론이다. <br>
> 컴파일된 IL에는 이미 구체적인 타입이 박혀있다.
>

## 추론 과정

### STEP 1 : 우변의 식 평가
우변의 표현식을 분석하여 타입 확정

### STEP 2 : 리터럴 기본값 적용

접미사 없는 리터럴은 언어 스펙의 기본 타입으로 결정

### STEP 3 : 타입 승격
복합 식은 암시적 변환 규칙에 따라 타입 승격

```csharp

var a = 10; // int
var b = 3.14 // double
var c = 1 + 2.0 // double(int 승격)
var d = GetUser() // 반환 타입 그대로   

```

## VAR VS DYNAMIC

| 항목 | `var` | `dynamic` |
|---|---|---|
| 타입 결정 시점 | 컴파일 타임 | 런타임 |
| 타입 안전성 | 보장 | 보장 안 됨 |
| IntelliSense | 완전 지원 | 제한적 |
| 잘못된 멤버 접근 | 컴파일 오류 | 런타임 예외 |
| 성능 오버헤드 | 없음 | 있음 (DLR) |

## 사용 불가 케이스
```csharp

var a; // 오류: 초기화 없이 선언 불가
var b = null; // 오류: null으로 타입 추론 불가

void Method(var n) {} // 오류: 매개변수에 사용 불가

```
